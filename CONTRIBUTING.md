# Contributing to Symphony Pro Studio

First off, thank you for considering contributing to Symphony! It's people like you that make Symphony such a great tool for the community.

## 🚀 How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker to report bugs.
- Describe the bug in detail and provide steps to reproduce.
- Include information about your macOS version and hardware if relevant.

### Suggesting Enhancements
- Enhancement suggestions are tracked as GitHub issues.
- Explain why this enhancement would be useful to most Symphony users.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code follows the existing style (Tailwind 4 + Framer Motion).

## 🎨 Design Guidelines
Symphony follows a **Pro Studio** aesthetic:
- Use glassmorphism where appropriate.
- Maintain high contrast with subtle glows.
- Prioritize fluid animations and tactile feedback.
- Stick to the established color palette (Black, White, Blue, Amber).

## 📜 Code of Conduct
Please be respectful and professional in all interactions within the Symphony community.
